# 04. Обзор fuzzing-инструментов

## Сравнительная таблица

|---|---|---|---|---|---|
| Hypothesis | Да | Property-based / generation-based | Простая интеграция с pytest, хорошая генерация edge cases, минимизация ошибок | Не является классическим coverage-guided fuzzer | Хорошо подходит |

## Hypothesis

`Hypothesis` — библиотека property-based testing для Python. Она позволяет описывать стратегии генерации данных и автоматически создавать множество тестовых случаев.

Применимость:

- генерация логинов;
- генерация паролей;
- проверка Unicode;
- проверка пустых строк;
- проверка длинных строк;
- проверка бинарных данных после декодирования.

Вывод: инструмент хорошо подходит для проекта.

## HypoFuzz

`HypoFuzz` — инструмент адаптивного fuzzing для property-based тестов. В контексте проекта он используется как coverage-guided слой поверх `Hypothesis` и `pytest`.

Применимость:

- coverage-guided запуск property-based тестов;
- проверка функции аутентификации;
- проверка функции хеширования;
- анализ покрытия кода через `pytest-cov`;
- проверка устойчивости к строковым и Unicode-входам.

Особенности:

- не является отдельным генератором входных файлов;
- не работает как black-box mutation fuzzer;
- не создаёт собственные crash-отчёты по аналогии с AFL или Radamsa;
- результаты фиксируются через pytest-логи и coverage-отчёты.

Вывод: инструмент хорошо подходит для текущего проекта как coverage-guided дополнение к Hypothesis, особенно с учётом того, что проект написан на Python и имеет небольшую тестируемую бизнес-логику.

## PythonFuzz

`PythonFuzz` — coverage-guided fuzzer для Python-кода.

Применимость:

- может использоваться для проверки функции аутентификации;
- может быть альтернативой `HypoFuzz`.

Ограничение:

- дублирует роль coverage-guided инструмента;
- может вызывать сложности совместимости с современными версиями Python и зависимостями;
- менее удобен для текущей среды тестирования.

Вывод: инструмент применим, но не выбран для основной практической части.

## Radamsa

`Radamsa` — языконезависимый mutation-based fuzzer. Он принимает seed-входы и создаёт их мутированные варианты.

Применимость:

- мутация строк `login:password`;
- мутация Unicode-входов;
- мутация специальных строк;
- black-box проверка устойчивости.

Вывод: инструмент подходит для проекта как mutation-based fuzzer.

## boofuzz

`boofuzz` предназначен для fuzzing сетевых протоколов.

Применимость:

- в текущей версии проекта сетевого интерфейса нет;
- API отсутствует;
- протокол обмена данными отсутствует.

Вывод: инструмент рассмотрен, но практически неприменим к текущей версии.

## Schemathesis

`Schemathesis` используется для fuzzing REST API и GraphQL API на основе схем.

Применимость:

- в проекте нет REST API;
- нет OpenAPI-схемы;
- нет GraphQL-схемы.

Вывод: инструмент неприменим к текущей версии проекта, но может быть полезен, если приложение будет расширено до клиент-серверной архитектуры.
