# 02. Постановка задачи фаззинга

## Цель фаззинг-тестирования

Цель fuzzing-тестирования — проверить устойчивость логики аутентификации приложения `python_auth` к некорректным, случайным, граничным и потенциально вредоносным входным данным.

## Объект тестирования

Объектом тестирования являются компоненты приложения, связанные с обработкой пользовательского ввода:

- логин пользователя;
- пароль пользователя;
- функция хеширования пароля;
- функция проверки пары логин/пароль;
- обработка неуспешной аутентификации;
- обработка количества оставшихся попыток входа.

## Основная задача fuzzing

Проверить устойчивость логики аутентификации к некорректным входным данным и убедиться, что приложение:

- не завершается аварийно;
- не выбрасывает необработанные исключения;
- не зависает;
- не допускает обход аутентификации;
- корректно обрабатывает пустые, длинные, Unicode и специальные строки.

## Формулировка задачи

Проверка устойчивости функций обработки логина и пароля к некорректным, случайно сгенерированным и мутированным входным данным.

## Типы входных данных

В рамках fuzzing проверяются следующие типы входов:

1. Валидные данные:
   - корректный логин;
   - корректный пароль.

2. Невалидные данные:
   - неправильный логин;
   - неправильный пароль;
   - пустой логин;
   - пустой пароль.

3. Граничные данные:
   - очень длинный логин;
   - очень длинный пароль;
   - строки максимальной длины;
   - строки из одного символа.

4. Unicode-данные:
   - кириллица;
   - китайские символы;
   - emoji;
   - смешанные строки.

5. Специальные строки:
   - строки с кавычками;
   - строки с управляющими символами;
   - SQL-like строки;
   - shell-like строки;
   - пути к файлам;
   - бинарные данные, декодированные как UTF-8.

## Ожидаемые классы ошибок

В ходе тестирования ожидается выявление следующих классов ошибок:

- необработанные исключения;
- падение приложения;
- зависание;
- чрезмерное потребление памяти;
- некорректная обработка Unicode;
- некорректная обработка пустых значений;
- некорректная обработка длинных строк;
- ошибка типа возвращаемого значения;
- потенциальный обход проверки логина и пароля;
- нарушение логики ограничения числа попыток входа.
