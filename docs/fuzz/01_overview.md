# 01. Общие сведения о приложении

## Описание проекта

Проект `python_auth` представляет собой простое Python-приложение аутентификации с графическим интерфейсом на `tkinter`.

Основная функциональность приложения:

- ввод логина;
- ввод пароля;
- проверка пары логин/пароль;
- хеширование пароля алгоритмом SHA-256;
- ограничение количества попыток входа;
- отображение результата аутентификации через GUI.

Приложение собирается в исполняемый файл с помощью `PyInstaller`. Также в проекте предусмотрена генерация документации через `Doxygen`.

## Особенности реализации

В текущей версии проекта бизнес-логика аутентификации находится в одном файле с GUI-кодом. Это усложняет автоматизированное тестирование, так как при импорте основного модуля создаётся окно `tkinter`.

Поэтому для фаззинг-тестирования целесообразно выделить или использовать отдельный тестовый слой, проверяющий не графический интерфейс, а внутреннюю логику:

- хеширование пароля;
- обработку логина;
- обработку пароля;
- сравнение хеша введённого пароля с эталонным хешем;
- обработку некорректных входных данных.

## Применимость fuzzing

Проект подходит для fuzzing, так как содержит обработку пользовательского ввода. Наиболее подходящими объектами фаззинг-тестирования являются функции, связанные с обработкой логина и пароля.

Прямой fuzzing GUI-слоя `tkinter` не является основным вариантом, так как:

- GUI требует графического окружения;
- в headless-среде приложение не запускается без `DISPLAY`;
- fuzzing GUI-событий сложнее и менее полезен для текущей версии проекта.

Основной акцент делается на fuzzing бизнес-логики аутентификации.
